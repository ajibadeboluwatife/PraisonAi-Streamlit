# Contributing to PraisonAi-Streamlit

We welcome contributions! Whether you're reporting an issue, suggesting a new feature, or submitting code changes, there are a few guidelines we'd like you to follow to make the process smooth and collaborative.

## Issue Reporting

- Clearly describe the issue with as much detail as possible.
- Include steps to reproduce the problem.
- Mention the system environment and setup (e.g., operating system, Python version, dependencies).
- Attach any relevant screenshots or code snippets.

## Feature Requests

- Explain the feature in detail.
- Provide examples of how the feature might be used, if applicable.
- Indicate why this would be beneficial to the project and its users.

## Pull Requests

- Create a separate branch for your changes.
- Keep your changes focused and relevant to the pull request's title and description.
- Add clear and concise commit messages that describe why a change was made.
- Ensure your code adheres to existing code styles and standards in the repository.
- Include tests for any new functionality or bug fixes, if necessary.
- Reference any related issues or feature requests using keywords (e.g., `Resolves #<issue_number>`).
- Be prepared to address review suggestions or feedback from maintainers.
